🏗️ Terraform Module Structure

Infrastructure is split into shared resources (one per account) and a reusable pipeline module (one instance per data source).

terraform/shared.tf
Shared infrastructure — created once, used by all pipelines
  • S3 data lake bucket (medallion layout)
  • KMS data key + PII key (auto-rotation)
  • IAM admin role + developer role
  • S3 bucket policy (deny raw PII to developer)
  • Glue database (stg_adobe)
  • Athena workgroup (adobe-stg, 100 MB limit)
terraform/modules/pipeline/
Reusable module — instantiated once per data source
  • IAM Lambda execution role (least-privilege)
  • Lambda function + EventBridge trigger
  • CloudWatch log group + error alarm
  • Glue tables: bronze_masked, bronze_raw, gold
  • S3 insert-overwrite permission for gold/
modules/adobe/terraform/pipeline.tf
Adobe-specific pipeline config — owned by the module, staged by CI
  • module "adobe_pipeline" → calls modules/pipeline
  • source_name = "adobe"
  • lambda_handler = "adobe.handler.lambda_handler"
  • 12 bronze columns (hit-level TSV schema)
  • 3 gold columns (aggregated, no PII)
terraform/observability.tf
Monitoring and cost management
  • CloudWatch dashboard (Lambda + Athena + KMS)
  • AWS Budgets (80% / 100% monthly alerts)
  • QuickSight dataset (optional, off by default)
terraform/
├── main.tf              # Provider + S3 backend config
├── variables.tf         # All variable declarations (project_name=adobe, env=stg)
├── shared.tf            # S3, KMS, IAM, Glue DB, Athena — shared across pipelines
├── observability.tf     # CloudWatch dashboard, Budgets, QuickSight
├── outputs.tf           # All root outputs
├── adobe_pipeline.tf    # ← staged from modules/adobe/terraform/pipeline.tf by CI
└── modules/
    └── pipeline/        # Reusable module (Lambda + EventBridge + Glue tables)
        ├── main.tf
        ├── variables.tf
        └── outputs.tf

modules/
└── adobe/               # Self-contained module: code + tests + Terraform config
    ├── src/adobe/       # Python package: handler.py, analyzer.py, dq_checker.py
    ├── tests/           # 32 unit tests (analyzer + DQ checker)
    └── terraform/
        └── pipeline.tf  # Adobe pipeline Terraform config (staged into terraform/ by CI)

🗂️ Data Layers

LayerS3 PrefixContentsRetentionGlue TableAccess
Landing landing/adobe/ Raw upload — triggers pipeline via EventBridge 60 days → delete — Lambda only
Bronze Raw bronze/raw/ Original hit-level data — plaintext ip + user_agent (PII) 365 days → Glacier adobe_bronze_raw Admin only
Bronze Masked bronze/masked/ SHA-256 hashed ip + user_agent — all other fields intact 365 days → Glacier adobe_bronze_masked Developer + Admin
Gold gold/ Aggregated keyword revenue — no PII, insert-overwrite on rerun 365 days → Standard IA adobe_gold Everyone
Athena Results athena-results/ Query output cache 7 days — Developer + Admin

🔐 Access Control Matrix

Resource Admin Role Developer Role Lambda Role
bronze/raw/ (S3) ✓ Read/Write ✗ Bucket policy Deny Write only
landing/ (S3) ✓ Read/Write ✗ Bucket policy Deny Read only
bronze/masked/ (S3) ✓ Read/Write ✓ Read Write only
gold/ (S3) ✓ Read/Write ✓ Read Write + List + Delete
PII KMS key (kms:Decrypt) ✓ Yes ✗ No access ✗ Encrypt only
adobe_bronze_raw (Glue/Athena) ✓ Full access ✗ Denied —
adobe_bronze_masked (Glue/Athena) ✓ Full access ✓ Read —
adobe_gold (Glue/Athena) ✓ Full access ✓ Read —

Developer access to bronze/raw is blocked at two independent layers: IAM Deny + S3 bucket policy Deny (cannot be overridden by any Allow except root).

🖥️ Quick Athena Queries

Run in Athena Query Editor → Workgroup: adobe-stg · Database: stg_adobe 

-- Top keywords by revenue (gold — no PII, anyone can query)
SELECT search_engine_domain, search_keyword, revenue
FROM stg_adobe.adobe_gold
ORDER BY revenue DESC;

-- Hit counts by page (masked bronze — developer accessible)
SELECT pagename, COUNT(*) AS hits
FROM stg_adobe.adobe_bronze_masked
GROUP BY pagename ORDER BY hits DESC;

-- Purchase events with pseudonymized IP (masked bronze)
SELECT date_time, ip, geo_city, product_list
FROM stg_adobe.adobe_bronze_masked
WHERE event_list LIKE '%1%';

-- Raw PII data — requires admin role + kms:Decrypt on alias/adobe-pii-stg
SELECT date_time, ip, geo_city, product_list
FROM stg_adobe.adobe_bronze_raw
WHERE event_list LIKE '%1%';

➕ Adding a New Data Source

The modular layout means adding a new source requires no changes to shared infra or the pipeline module — only new files in a new module directory.

# Step 1: Create the new module directory
cp -r modules/adobe modules/salesforce

# Step 2: Rename the Python package and update handler logic
mv modules/salesforce/src/adobe modules/salesforce/src/salesforce
# Edit modules/salesforce/src/salesforce/handler.py — update transformation logic

# Step 3: Update modules/salesforce/terraform/pipeline.tf
module "salesforce_pipeline" {
  source = "./modules/pipeline"

  source_name    = "salesforce"
  lambda_handler = "salesforce.handler.lambda_handler"
  bronze_columns = [
    { name = "contact_id", type = "string",  comment = "" },
    { name = "revenue",    type = "double",   comment = "" },
  ]
  gold_columns = [
    { name = "campaign", type = "string", comment = "" },
    { name = "revenue",  type = "double", comment = "" },
  ]
  # All shared infra wired automatically — copy remaining vars from adobe block
  project_name          = var.project_name
  environment           = var.environment
  ...
}

# Step 4: Push a PR
# CI automatically:
#   - Runs module tests (modules/salesforce/tests/)
#   - Packages salesforce code into the shared lambda.zip
#   - Stages modules/salesforce/terraform/pipeline.tf → terraform/salesforce_pipeline.tf
#   - Runs terraform plan → posts diff to PR
#   - On merge: terraform apply → new Lambda + Glue tables + EventBridge rule live
